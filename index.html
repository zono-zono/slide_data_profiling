データ品質って何？

データ品質とは？

データがビジネスに有用な形で活用できる状態かどうかを表す指標。

  • 正確で信頼できるデータ
  • 欠損や不整合がない
  • 必要なときに使える
  • 意図通りに意味が通じる

難しさ・よくある悩み

  • そもそも「データ品質」が何を指すのか曖昧
  • どの項目から見てよいか分からない
  • 改善のゴールが見えづらい
  • 関係者間で認識がバラバラ
まずは「整合性」「欠損」「型」「一意性」など、基本項目から着手するのがおすすめ。

実装例：画像データのプロファイリング

画像データプロファイリングの重要性

画像データのメタデータや画質、解像度、色空間を確認することで、データの一貫性や利用可能性を評価します。

Python実装

from PIL import Image
import os

def profile_image_folder(folder_path):
    resolutions = []
    file_sizes = []
    modes = []

    for filename in os.listdir(folder_path):
        if filename.lower().endswith(('.png', '.jpg', '.jpeg')):
            img_path = os.path.join(folder_path, filename)
            img = Image.open(img_path)
            resolutions.append(img.size)
            file_sizes.append(os.path.getsize(img_path) / 1024)  # KB
            modes.append(img.mode)

    return resolutions, file_sizes, modes

resolutions, file_sizes, modes = profile_image_folder('/path/to/images')

print("Resolutions:", resolutions)
print("File Sizes (KB):", file_sizes)
print("Color Modes:", modes)

実装のポイント

  • 画像解像度の分布を確認
  • ファイルサイズのばらつきを確認
  • 色空間（RGB, Grayscale など）の整合性を確認

レコード数検証の重要性

データセットの総レコード数を確認し、期待値との差異を検出します。このチェックにより、データの欠落や重複を早期に発見できます。

Python実装

def check_record_count(df, expected_count=None):
    actual_count = len(df)
    diff_rate = None
    if expected_count is not None:
        diff_rate = abs(actual_count - expected_count) / expected_count
    return actual_count, diff_rate

# 関数の使用例
record_count, diff_rate = check_record_count(df, expected_count=10000)
print("Record count:", record_count, "Difference rate:", diff_rate)

# レコード数の比較を可視化
plt.figure(figsize=(10, 5))
plt.bar(['Actual', 'Expected'], [record_count, 10000])
plt.title('Record Count Comparison')
plt.ylabel('Number of Records')
plt.show()

実装のポイント

  • 期待値との差を割合（%）で評価
  • 期待値が未設定の場合も対応可能
  • 可視化によりデータ量の差を直感的に把握

完全性／充足率の重要性

必須フィールドのデータが正しく入力されているかを評価します。データの欠損状況を把握し、分析や意思決定の信頼性を確保するための基本指標です。

Python実装

def check_completeness(df, required_fields=None):
    if required_fields is None:
        required_fields = df.columns
    
    # 必須フィールドのNULL値をチェック
    null_counts = df[required_fields].isnull().sum(axis=1)
    complete_records = (null_counts == 0).sum()
    completeness_ratio = complete_records / len(df)
    
    # カラムごとのNULL値の割合
    column_null_ratios = df[required_fields].isnull().mean()
    
    return complete_records, completeness_ratio, column_null_ratios

# 必須フィールドを指定して完全性をチェック
required_fields = ['vendor_id', 'pickup_datetime', 'dropoff_datetime', 
                  'passenger_count', 'trip_distance', 'fare_amount']
complete_records, completeness_ratio, column_null_ratios = check_completeness(df, required_fields)

# 結果の可視化
plt.figure(figsize=(12, 6))
plt.subplot(1, 2, 1)
plt.bar(['Complete', 'Incomplete'], 
        [complete_records, len(df) - complete_records])

実装のポイント

  • 必須フィールドを明示的に指定可能
  • レコード単位とフィールド単位の2つの視点から評価
  • ヒートマップでNULL値の分布パターンを視覚化

実装例3：NULL値チェックの解説

NULL値検出の重要性

NULL値はデータの欠損を表し、分析結果に偏りをもたらす可能性があります。NULL値の分布を把握することで、データの品質向上のための優先順位付けが可能になります。

Python実装

def check_null_values(df):
    # NULL値の数を集計
    null_counts = df.isnull().sum()
    # NULL値の割合を計算
    null_ratios = df.isnull().mean()
    
    null_stats = pd.DataFrame({
        'null_count': null_counts,
        'null_ratio': null_ratios
    })
    
    return null_stats

null_stats = check_null_values(df)
print("NULL Statistics:\n", null_stats)

# NULL値の分布を可視化
plt.figure(figsize=(12, 6))
sns.heatmap(df.isnull(), yticklabels=False, cbar=True, cmap='viridis')
plt.title('NULL Values Distribution')

# 各カラムのNULL値の割合を棒グラフで表示
plt.figure(figsize=(10, 5))
sns.barplot(x=null_stats['null_ratio'], y=null_stats.index)
plt.title("NULL Ratio per Column")

NULL値への対処法

  • 削除：欠損が少ない場合、レコードまたは列を削除
  • 補完：平均値・中央値・最頻値などで埋める
  • モデリング：回帰モデルなどで予測値を算出
  • 特殊値：NULL専用の特殊値（-1など）で置換

NULL値の可視化

可視化のポイント

  • ヒートマップ：NULL値の分布パターンを直感的に確認できる
  • 棒グラフ：カラムごとのNULL率を定量的に比較できる
  • 相関分析：複数カラムでのNULLの関連性を発見できる

データプロファイリング実装例4：内容適合性

実装例4：内容適合性のチェック

内容適合性の重要性

内容適合性は、データ値がビジネスルールや条件に適合しているかを検証します。例えば、人数は正の値であるべき、料金は0より大きいべきなど、論理的制約に基づく検証です。

Python実装

def check_content_suitability(df, rules_dict):
    # ルール辞書に基づいてデータの適合性をチェック
    results = {}
    for column, rule in rules_dict.items():
        if column in df.columns:
            # 各カラムに対してルールを適用し、適合率を計算
            results[column] = df[column].apply(rule).mean()
    return results

# ビジネスルールの定義
rules = {
    'passenger_count': lambda x: 0 < x,
    'trip_distance': lambda x: x > 0,
    'fare_amount': lambda x: x > 0
}
suitability_results = check_content_suitability(df, rules)
print("Content Suitability Results:", suitability_results)

# 内容適合性の結果を可視化
unsuitable_ratios = {col: 1 - ratio for col, ratio in suitability_results.items()}
plt.figure(figsize=(10, 5))
plt.bar(unsuitable_ratios.keys(), unsuitable_ratios.values())
plt.title('Ratio of Records Not Meeting Content Rules')
plt.xlabel('Column')
plt.ylabel('Unsuitable Ratio')

実装のポイント

  • lambda関数でルールを簡潔に表現
  • 複雑なルールは別途関数として定義可能
  • 各ルールの適合率を計算して比較
  • 不適合データは個別に詳細調査が必要

内容適合性の可視化

一般的な問題例

  • 負の値の利用料金
  • 0人の乗客数
  • 空の移動距離（0 km）
  • 不可能な時間値

対処法

  • ビジネスルールの明確化
  • 入力フォームのバリデーション強化
  • 自動補正ルールの設定
  • 異常値の通知システム構築
